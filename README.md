# Latihan 21-23: Final Project - Pembuatan Website Statis

Waktunya membuktikan semua ilmu yang sudah kamu pelajari (HTML, CSS, Git, JS)! Proyek ini adalah karya monumental kamu sebelum kita masuk ke dunia PHP (Backend).

## Tujuan Utama
Membuat website statis yang responsif, interaktif, dan dideploy dengan benar. Website ini harus memiliki minimal 3 halaman yang saling terhubung.

---

## 💡 Ide Proyek (Pilih Salah Satu)

1.  **Personal Portfolio 2.0**: Versi upgrade dari Resume yang sudah kamu buat. Harus ada galeri project (Grid/Flexbox) dan form contact yang tervalidasi JS.
2.  **Landing Page Bisnis Sederhana**: Contoh: Website Kedai Kopi, Barbershop, atau Toko Online statis. Harus ada katalog produk dan fitur slider (Carousel).
3.  **Halaman Berita/Artikel**: Portal berita simpel bertema hobi (misal: Game, Masak, atau Gadget). Harus ada daftar artikel dan fitur cari (filter statis JS).
4.  **Aplikasi Web Interaktif**: Misal: To-Do List, Dashboard Keuangan Statis, atau Quiz Game sederhana.

---

## ✅ Kriteria Penilaian

1.  **Struktur (HTML)**: Gunakan tag semantik (<header>, <nav>, <main>, <article>, <footer>). Gak boleh asal pakai <div> ya!
2.  **Tampilan (CSS Flexbox)**: Layout harus rapi dan terlihat profesional. Gunakan Flexbox untuk navigasi dan konten kolom.
3.  **Responsivitas (Media Queries)**: Website harus enak dilihat di HP dan Laptop. Minimal ada 1 breakpoint media query.
4.  **Interaktivitas (JavaScript)**: Harus ada minimal satu fitur JS (Contoh: Modal, Slider, Dark Mode, atau Validasi Form).
5.  **Manajemen Kode (Git/GitHub)**: Commit harus bermakna dan repository harus rapi.
6.  **Deployment**: Website harus live di GitHub Pages atau Netlify.

---

## 📅 Rundown Pengerjaan

*   **Hari 1 (Pertemuan 21)**: Perencanaan (Mockup) dan Struktur Dasar (HTML).
*   **Hari 2 (Pertemuan 22)**: Styling (CSS) dan Interaksi (JS).
*   **Hari 3 (Pertemuan 23)**: Polishing, Bug Fixing, Deployment, & Showcase (Presentasi).
